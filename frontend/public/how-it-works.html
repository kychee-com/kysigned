How it works

Kysigned turns a signer email forward into a verifiable signature by relying on DKIM, the cryptographic signature email providers already add to outbound mail.

  1. The creator sends a PDF. Kysigned records the document hash, signers, and signing instructions.
  2. Each signer forwards the request email. The first line states the signing intent, and the original PDF stays attached.
  3. The signer email provider signs the forward. DKIM proves the message came from that domain's mail system and has not been changed.
  4. Kysigned checks the evidence. It verifies the DKIM signature, signer address, signing phrase, attachment bytes, timestamps, and envelope state.
  5. Everyone receives a bundle. The final PDF carries the original document plus enough evidence for later offline verification.

Try it before sending anything: the repository ships sample files under docs/test-assets/, including a genuine signed record for the verify page and matching originals for the hash-check tool, each with a documented expected result.

Running your own instance? Clone the repo, set KYSIGNED_ALLOWED_CREATORS, and run run402 up --name your-name. Your instance gets its own web origin and project mail host.
For technical verification details, see the repository trust-model documentation.
